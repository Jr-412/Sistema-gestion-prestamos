Actúa como un desarrollador Full Stack Java + React que está realizando una prueba técnica con un límite estricto de 3 horas.
No quiero una solución idealizada ni un proyecto con características propias de varios días de desarrollo. Quiero una implementación realista, como la que entregaría un buen desarrollador al finalizar una prueba técnica en ese tiempo.
Contexto
La aplicación corresponde a una prueba técnica de Spring Boot + React.
Debes respetar estrictamente los requerimientos del documento.
Sin embargo, debes priorizar la entrega de un sistema completo, funcional, limpio y bien organizado, antes que intentar implementar absolutamente todas las recomendaciones del documento.
La prioridad es demostrar conocimientos técnicos y buenas prácticas, no construir un producto listo para producción.

Qué simplificar por tiempo
Dado que esta es una prueba de aproximadamente tres horas, puedes simplificar razonablemente algunos aspectos siempre que la funcionalidad principal esté completa.
Por ejemplo:
Utilizar Context API en lugar de Redux.
Implementar DTOs únicamente donde aporten claridad.
Mantener una arquitectura por capas en lugar de una hexagonal completa.
Crear un conjunto pequeño de pruebas unitarias para la lógica más importante.
Implementar una solución de caché sencilla (por ejemplo, Caffeine con @Cacheable) si el tiempo lo permite; si no, dejar la configuración preparada y documentar que fue omitida por limitaciones de tiempo.
Estas simplificaciones deben ser coherentes con un desarrollo realizado bajo presión y no afectar el funcionamiento principal.
